# プッシュ前確認チェックリスト

機能追加・修正後、プッシュ前にすべて確認する。
チェックが通らない項目があれば修正してからプッシュすること。

---

## 0. 静的チェック（自動）

```bash
npx tsc --noEmit
```

- [ ] エラーなしで通る

---

## 1. サイト側（公開ページ）

### 未ログイン状態

- [ ] トップページ（LP）が表示される
- [ ] 料金ページが表示される（CTA が「無料で始める」「月額プランを始める」）
- [ ] LP のよくある質問が複数同時に開けられる
- [ ] FAQ ページ下部のお問い合わせが `/contact` に遷移する（メーラーが起動しない）
- [ ] キャスト紹介ページが表示される
- [ ] Cast Talk 一覧・詳細が表示される
- [ ] ブログ一覧・詳細が表示される
- [ ] 利用規約・プライバシーポリシー・特商法が表示される

### ログイン済み状態

- [ ] LP の CTA が「ダッシュボードへ」に変わる（「無料で始める」が出ない）
- [ ] 料金ページの契約中プランに「現在ご契約中」バッジが出る
- [ ] 契約中プランのチェックアウトボタンが押せない（グレーアウト）

### 認証フロー

- [ ] メール＋パスワードで新規登録できる（確認メールが届く）
- [ ] Google で新規登録できる
- [ ] メール＋パスワードでログインできる
- [ ] Google でログインできる
- [ ] ログアウトできる
- [ ] パスワードリセットメールが届く・再設定できる
- [ ] `?plan=personal` 付きでサインアップ → メール認証後に Stripe チェックアウトへ遷移する

---

## 2. ツール側（ログイン後）

### ダッシュボード

- [ ] ダッシュボードが表示される
- [ ] 未認証でアクセスするとログインページにリダイレクトされる

### 取材先

- [ ] 取材先一覧が表示される
- [ ] 取材先を新規作成できる
- [ ] 無料・個人プランで 2 件目を作ろうとすると上限メッセージが出る
- [ ] プランダウングレード後、ロック中バナーが一覧に表示される
- [ ] ロック中の取材先カードに「取材不可」が表示される（「取材する」ボタンがない）

### インタビュー

- [ ] キャスト選択ページが表示される
- [ ] テーマ選択・自由入力・お任せのいずれかでインタビューを開始できる
- [ ] インタビュー会話が送受信できる（AI が返答する）
- [ ] 無料プラン上限（生涯 2 回）到達後にブロック画面が出る
- [ ] 月間上限到達後にブロック画面が出る
- [ ] ロック中取材先のインタビューページを開くとブロック画面が出る

### 記事・設定

- [ ] インタビュー後に記事生成ができる
- [ ] 記事テキストをコピー・ダウンロードできる
- [ ] プロフィール編集できる（名前の保存が反映される）
- [ ] 設定 → 支払い管理から Stripe Customer Portal が開ける

---

## 3. 課金（Stripe テストモード限定）

> テストカード: `4242 4242 4242 4242`（有効期限・CVV は任意）

- [ ] 未契約ユーザーが個人プランを購入できる
- [ ] 同じプランを再購入しようとするとエラーが出てチェックアウトに遷移しない
- [ ] 上位プランのユーザーが下位プランを購入しようとするとエラーが出る
- [ ] 購入完了後に `/settings/billing?success=1` へ遷移する
- [ ] webhook 受信後に `subscriptions` テーブルのプランが更新される（admin で確認）

> ⚠️ 本番カードは絶対に使わない

---

## 4. admin 側

> `/admin` は `ADMIN_EMAILS` に登録されたメールでのみアクセス可

- [ ] 管理者メールでのみ `/admin` にアクセスできる（非管理者はリダイレクトされる）
- [ ] ユーザー一覧が表示される
- [ ] ユーザーのプランを変更できる（変更が即時反映される）
- [ ] メール確認なしでユーザーを作成できる（作成したアカウントで即ログインできる）
- [ ] Cast Talk 一覧・詳細・評価フォームが表示される

---

## 関連ルール

- **課金・認証・プラン判定の変更後は必ずレビュアーに確認させる**（CLAUDE.md ガードレール 12 条）
- Stripe の確認はテストモード（`sk_test_` キー）で行う
