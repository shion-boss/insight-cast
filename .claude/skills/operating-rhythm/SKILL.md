# Operating Rhythm

## 目的
チーム運用を個人技にせず、週次と月次の確認を同じ型で回す。

## 使う場面

- `ops/weekly-review.md` を更新する時
- 中間マイルストーンの進捗を整理する時
- 月次コストの要点をまとめる時

## 週次で見ること

- 今週進んだこと
- 止まっていること
- 意思決定が必要なこと
- 来週やること

## 月次で見ること

- コストの増減
- モード逸脱の有無
- 成果指標の変化

## 出力フォーマット

```md
## 定例まとめ

### 今回の対象期間
- ...

### 進んだこと
- ...

### 詰まり
- ...

### 要判断
- ...

### 次の打ち手
1. ...
2. ...
```

## 注意

- 事実と解釈を混ぜない
- 数字があるものは数字で書く
- 次アクションがない定例まとめにしない
