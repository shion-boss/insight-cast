# Competitor Analysis Template

## 目的
競合サイトを比較して、  
**自社に足りない情報** と **参考にすべき見せ方** を整理する。

この skill は、以下のような時に使う。

- 顧客の競合URLを比較する時
- 競合が強く出している要素を見つけたい時
- 自社に不足している発信テーマを探したい時
- 発信テーマ提案や改善提案を作る時

## 基本方針
競合分析は、勝ち負け判定ではない。  
**相対比較によって、自社がまだ出せていない価値を見つけること**が目的。

## 比較する観点

### 1. 誰向けか
- 競合は対象顧客を明確にしているか
- 地域や用途をはっきり出しているか
- 自社より分かりやすいか

### 2. サービスの見せ方
- 提供内容が分かりやすいか
- 初見で理解しやすいか
- 導入のしやすさが見えるか

### 3. 強みの見せ方
- 競合は何を強みとして出しているか
- その強みは具体的か
- 自社は何をまだ出せていないか

### 4. 不安解消の量と質
- FAQ
- 料金説明
- 流れ
- 注意点
- 初めて向け説明

競合の方が親切なら、その差を取る。

### 5. 実績・信頼材料
- 施工事例 / 実績
- 口コミ
- スタッフ紹介
- 写真
- 数字
- 地域密着感

### 6. 記事・ブログ・お知らせ
- 何をテーマに更新しているか
- どのくらい継続しているか
- 顧客の不安解消になっているか
- 一般論か一次情報か

### 7. CTAと導線
- 問い合わせしやすいか
- CTAが自然に見えるか
- スマホでも分かりやすいか

### 8. 会社らしさ
- 競合は「人」が見えるか
- 空気感や姿勢が見えるか
- 自社には何が足りないか

### 9. 検索 / AI時代に強そうか
- 一次情報があるか
- 比較検討に必要な材料があるか
- AIに要約されても差が残るか

### 10. 真似すべきこと / 真似しなくていいこと
- 参考になる点
- 表面的に真似しても意味がない点
- 自社に合わせて取り入れるべき点

## 分析時の注意
- 単に「競合の方が良い」と言わない
- 自社に活かせる形に変換する
- 真似ではなく、補強のヒントとして使う
- 強みを奪うのでなく、自社にない観点を見つける
