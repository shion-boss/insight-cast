# インタビュー品質評価ルーブリック

作成: ai-architect / 運用・改訂: reviewer

---

## 評価対象

Insight Cast のプロダクトAI（インタビュアーキャラ）が生成したインタビューセッション全体を評価する。

---

## 採点方式

**100点満点 / 5軸 × 各20点**

| 点数 | 判定 |
|---|---|
| 85点以上 | 優良 — そのまま記事・LP素材として使える |
| 70〜84点 | 合格 — 軽い編集で使える |
| 55〜69点 | 要改善 — 再インタビューまたは追加質問が必要 |
| 54点以下 | 不合格 — 根本的な設計の見直しが必要 |

---

## 評価軸

### 軸1: 構えさせない設計（20点）

事業者が「当たり前」を語れる心理状態を作れているか。

| 点数 | 基準 |
|---|---|
| 17〜20 | 最初の質問が強み・実績の直撃になっていない。雑談や身近な話題から入っている。事業者が「答えやすそう」と感じる切り出し方ができている |
| 12〜16 | おおむね構えさせない設計だが、一部に圧を与える質問がある |
| 7〜11 | 「御社の強みは？」「差別化ポイントは？」のような直球質問が序盤にある |
| 0〜6 | 全体的に圧が強く、事業者が防御的になっている |

### 軸2: 当たり前の価値の言語化（20点）

事業者自身が気づいていない・語っていない価値を引き出せているか。

| 点数 | 基準 |
|---|---|
| 17〜20 | 事業者が「そういえばそうですね」「言われてみれば」と気づく瞬間が1つ以上ある。他社との違いが事業者の言葉で出ている |
| 12〜16 | 多少の新しい気づきはあるが、既知の情報の確認にとどまっている部分が多い |
| 7〜11 | 事業者が既に語っていることをなぞっているだけ |
| 0〜6 | 引き出しがなく、一般論・建前の回答のみ |

### 軸3: HP強化に使える一次情報の量と質（20点）

固有のエピソード・数字・お客様の声が記事・LP素材として使えるレベルで出ているか。

| 点数 | 基準 |
|---|---|
| 17〜20 | 固有のエピソードが2つ以上、具体的な数字（年数・件数・距離・時間等）が1つ以上、お客様の声またはリアクションの描写がある |
| 12〜16 | 固有のエピソードは1つあるが、数字や具体描写が薄い |
| 7〜11 | エピソードが抽象的で記事に使いにくい |
| 0〜6 | 一般論のみで固有情報がほぼない |

### 軸4: 会話の自然な流れ（20点）

一問一答になっていないか。前の回答を受けて掘り下げられているか。

| 点数 | 基準 |
|---|---|
| 17〜20 | 事業者の回答の中のキーワードを拾って次の質問に繋げている。会話として読める流れになっている |
| 12〜16 | 一部で前の回答を受けているが、途中で流れが途切れるところがある |
| 7〜11 | ほぼ一問一答。質問リストを順番に消化しているだけ |
| 0〜6 | 回答を全く受けていない。文脈のない質問が続く |

### 軸5: キャラの一貫性（20点）

設計書（docs/ai-specs/）が定めるキャラの引き出す観点・言葉遣い・入力方式が守られているか。

| 点数 | 基準 |
|---|---|
| 17〜20 | そのキャラが担当する観点（例: ミントならお客様目線・安心感）に沿った質問設計になっている。言葉遣いが設計書の方針と一致している |
| 12〜16 | 観点はおおむね合っているが、他キャラの領域に踏み込んでいる部分がある |
| 7〜11 | キャラの個性が薄く、どのキャラでも同じ質問になっている |
| 0〜6 | 設計書の観点と大きくズレている |
