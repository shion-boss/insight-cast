# Copywriting for LP / Web Marketing

## このスキルを使う場面

- LP・ランディングページのコピーを書く・レビューする時
- ブログ・SNS投稿など流入コンテンツのコピーを設計する時
- ヘッドライン・CTAボタン・ペインセクションを磨く時
- Insight Cast のサービスコピーを書く・批評する時（マーケター）
- プロダクトAIが生成する記事・要約テキストの質を評価する時（AIデザイナー）

---

## 1. フレームワークの選び方

| 状況 | 推奨 |
|---|---|
| 認知フェーズの広告・SNS短尺 | PAS |
| 悩み解決型LP（BtoC） | PASONA |
| 高単価・信頼証明が必要 | AIDCA |
| 変化・体験を売るコンテンツ | BAB |
| 機能説明が多いBtoB | FAB |
| LP全体の骨格設計 | AIDCA or PASONA |

---

## 2. フレームワーク詳細

### PAS（最短・最強の構文）
```
Problem（問題提起） → Agitate（痛みを増幅） → Solution（解決策）
```
**使いどころ**: メール件名・SNS・バナー・ブログリード

**例（Insight Cast）**:
> HPを更新したいのに、何を書けばいいか分からない。
> そのまま放置すると、競合との差はじわじわ広がるばかり。
> Insight Castのインタビューが、あなたの中にある答えを引き出します。

**よくある失敗**:
- 問題提起がない → 読者に「自分のことだ」と思わせられない
- 解決策が自社機能の説明で終わる → 読者が得るものに変換していない

---

### PASONA（日本LP定番）
```
Problem → Agitation → Solution → Offer → Narrow down → Action
```

| ステップ | 目的 | 文量 |
|---|---|---|
| Problem | 「あなたの問題ですよね」と共感 | 2〜3文 |
| Agitation | 放置するとどうなるかを見せる | 2〜3文（あおりすぎない） |
| Solution | 解決の方向性を提示 | 1〜2文 |
| Offer | 具体的なサービス・価格・内容 | セクション全体 |
| Narrow down | 「誰向けか」「今がタイミングな理由」 | 1〜2文 |
| Action | CTA | ボタン＋補足1文 |

**注意**: Agitationは2〜3文で留める。長くなると「脅している」印象になり日本人には逆効果。

---

### AIDCA（高単価・信頼重視）
```
Attention → Interest → Desire → Conviction（確信） → Action
```

**AIDAとの違い**: Conviction（確信）を挟む。実績・事例・第三者評価がここに入る。

**LPセクションへのマッピング**:
- Attention → FVヘッドライン
- Interest → ペイン・課題共感セクション
- Desire → ベネフィット・仕組み説明
- Conviction → 実績・口コミ・メディア掲載・FAQ
- Action → クロージングCTA

---

### BAB（変化・体験を売る）
```
Before（現状の痛み） → After（理想の状態） → Bridge（その方法）
```

**Insight Cast 使用例**:
> HPは持っているのに、最後に更新したのがいつか思い出せない。（Before）
> 3ヶ月後、インタビューで引き出した自社の強みが記事になり、
> 初めて検索から問い合わせが来た。（After）
> Insight Castは、あなたの中にある答えを引き出すことから始めます。（Bridge）

**原則**: 変化の主体は常に読者。「弊社がやってあげた」ではなく「あなたが手に入れた」の構文で書く。

---

### FAB（機能→価値の変換）
```
Feature（機能） → Advantage（優位性） → Benefit（顧客が得るもの）
```

**例（Insight Cast）**:
- Feature: AIインタビュー機能
- Advantage: 聞き方をAIが最適化するので、質問を自分で考えなくていい
- Benefit: あなたが気づいていなかった強みが、そのまま記事の言葉になる

**失敗パターン**: Feature止まり → 「AIを使ったインタビュー機能を搭載しています」（読者の利益が見えない）

---

## 3. LP構成の黄金律

### セクション順序

```
1. FV（ヘッドライン＋キービジュアル＋CTA）
      ↓ 共感・問題定義
2. ペインセクション
      ↓ 解決の方向性
3. サービス概要・仕組み
      ↓ 信頼構築
4. 実績・社会的証明（数字・事例・口コミ・メディア）
      ↓ 具体的な価値
5. ベネフィット列挙（3〜5つ）
      ↓ 疑問・懸念の解消
6. FAQ
      ↓ リスク軽減
7. 保証・解約条件・サポート体制
      ↓ 最終判断
8. 料金・プラン
      ↓ 行動
9. クロージングCTA
```

### CTA の3点配置（最低限）
1. **FV内** — 「まず見てみる」程度の低摩擦な文言
2. **ベネフィット直後** — 「これを得たいと感じた瞬間」に
3. **ページ末尾** — 読み切った人への最終呼びかけ

## 4. ヘッドライン（見出し）の型

### 6つの型

| 型 | 例 |
|---|---|
| **数字＋ベネフィット** | 「3つの問いに答えるだけで、HPに載せるテーマが決まります」 |
| **ターゲット明示＋共感** | 「HP更新に悩む工務店オーナーへ。強みを言葉にする方法、ありました」 |
| **ビフォアアフター圧縮** | 「書くことが思いつかなかったHPが、取材の言葉で動き始めた」 |
| **問いかけ** | 「あなたのお店の"当たり前"、まだ伝わっていませんよね？」 |
| **逆説・意外性** | 「ライターを雇わなくても、HPが育つ理由」 |
| **方法提示** | 「30分のインタビューで、HPを強くするネタを引き出す方法」 |

### 数字の使い方
- 「短時間で」→ 弱い
- 「30分で」→ 強い（具体的）
- 「多くのお客様に」→ 弱い
- 「47社の中小企業が導入」→ 強い（数字＋属性）
- 奇数のほうが「作られた感」が薄れる（10より11、5より7）

### ターゲット絞り込みの段階

| 粗い | 中程度 | 精密 |
|---|---|---|
| 「中小企業のオーナーへ」 | 「HPを持つ飲食店オーナーへ」 | 「開業3〜10年、HP更新が止まっている飲食店オーナーへ」 |

精密なほど「自分のことだ」と感じる人が増える。ターゲット外の人が離脱するのは正しい動作。

## 5. CTA コピーの原則

### ボタンテキスト設計の3原則

**原則1: 読者を主語にする**
- NG: 「申し込む」「送信する」（操作的）
- OK: 「無料で試してみる」「インタビューを体験する」

**原則2: 行動後の状態を見せる**
- NG: 「登録する」
- OK: 「登録してHPのヒントをもらう」

**原則3: リスク除去をボタン直下に置く**
```
[無料で始める]
クレジットカード不要・いつでも解約可能・初期費用0円
```

### 緊急性・限定性（やりすぎに注意）

日本語コピーでは、根拠のない緊急性は逆効果になりやすい。

- OK: 「今なら初月50%OFF（〇月〇日まで）」← 期限は必ず守る
- OK: 「先着30名限定の無料相談」← 根拠があれば有効
- NG: 「残りわずか！今すぐ！」← 根拠なし → 信頼を損なう

緊急性より「このタイミングで始める合理的な理由」を示す方が日本語コピーでは信頼が上がる。

## 6. 日本語コピーの特性

### 響く表現パターン

**責めない共感**
> 「毎日お客様対応で忙しい中、HP更新まで手が回らないのは当然です」

**権威・第三者証明**（日本人は横並び意識と権威への信頼が強い）
- 「Yahoo!ニュースに掲載」「〇〇賞受賞」「顧客満足度94%」
- FVの近くに掲載バナーを置く

**「みんな使っている」安心感**
> 「全国47都道府県・1,200社が導入」

**「人が関与している」安心感**
> 「ご不明な点はいつでもチャットでご相談ください（担当者が直接お答えします）」

### 信頼構築の要素（優先順）
1. 具体的な数字（「多数」でなく「127社」）
2. 顔出し・実名入りの声
3. 受賞・メディア掲載
4. 企業情報（代表者名・所在地）
5. 失敗事例・向かないケースの正直な開示 ← 「誠実さ」が日本語コピーでは特に効く

### 不安解消の書き方
- 「もし合わなければ、いつでも解約できます（手続き5分）」
- 「初期費用ゼロ、最低契約期間なし」
- 「ITが苦手でも、サポートがあるから大丈夫です」
- FAQは「読者が感じる不安のリスト」として設計する

## 7. ペイン言語化の技術

### 「自分のことだ」と感じさせる4要素

1. **属性の絞り込み**: 「飲食店オーナー」より「開業3〜10年・スタッフ5名以下の飲食店オーナー」
2. **内心の描写**: 「〜と思いながら」「〜なのに〜してしまう」
3. **行動パターンの描写**: 「SNSを開いては閉じ、ブログを書こうとして白紙のまま」
4. **言い訳・合理化の引用**: 「忙しいから仕方ない、と思っているその気持ち」

### ペイン発見の4つの問い
1. 読者が夜中に気になっていることは何か？
2. 「もしこれができたら…」と思っていることは何か？
3. 過去に試みて失敗したことは何か？
4. 周りに相談しづらくて一人で抱えていることは何か？

### ペイン言語化の良例・悪例

| 悪例（自社目線） | 良例（読者の言葉） |
|---|---|
| 「SEO対策が不十分な状態」 | 「"そのうち更新しよう"と思ったまま、また今日も手つかずのまま」 |
| 「コンテンツ不足」 | 「競合のHPを見るたびに焦るのに、自社の強みをうまく言葉にできない」 |
| 「更新頻度が低い」 | 「最後に更新したのがいつか、もう思い出せない」 |

## 8. サービス系・抽象的商品の具体化

### 3つの技法

**技法1: アウトプットで語る**
- 抽象: 「HP改善のサポートをします」
- 具体: 「インタビュー1回で、HPに掲載できる記事素材が5〜8本分生まれます」

**技法2: プロセスを見せる（ステップ図）**
```
① URLを入れる
② AIが競合と自社を比較する
③ 差別化ポイントがリストで出る
④ インタビューで深掘りする
```

**技法3: 使用場面の具体描写**
> 「朝30分のインタビューが終わったら、そのまま会議に向かえます」
> 「スマホから話すだけで、翌日には記事の素材ができています」

### ROIの示し方
- **コスト比較**: 「月額4,980円で、外注ブログ代（1本2〜5万円）が不要に」
- **時間ROI**: 「取材から記事化まで2週間かかっていた作業が3日に短縮」
- **実績ROI**: 「3ヶ月で問い合わせ数が1.4倍になった事例」

## 9. Insight Cast 専用コピーパターン集

### ヘッドライン候補

**共感・逆説型**
- 「HPを更新できていないのは、ネタがないからではなかった」
- 「書けないのではなく、引き出せていなかっただけかもしれない」

**世界観型（アイコンと組み合わせる）**
- 「動物AIインタビュアーが、取材に来ます」
- 「あなたの当たり前を言葉にする、AIの会社があります」

**ベネフィット直球型**
- 「30分話すだけで、HPを強くするネタが5本以上出てくる理由」

### ペインセクション候補
- 「更新しなければと思いながら、何を書けばいいか分からない」
- 「外注すると高い。かといって自分で書く時間もない」
- 「競合のHPを見ると焦るのに、自社の強みをうまく言葉にできない」
- 「SEOがどうとか言われても、そもそも何を書けばいいかが分からない」

### CTA候補

| 場面 | 推奨文言 |
|---|---|
| FV（低摩擦） | 「まず1回、インタビューを試してみる（無料）」 |
| 料金後 | 「個人プランで始める（月4,980円・解約自由）」 |
| 末尾クロージング | 「あなたのHPを一緒に育てましょう」 |
| ログイン済みユーザー向け | 「ダッシュボードへ」 |

### ターゲット絞り込みパターン
- 「HPはあるのに、最後に更新したのが半年以上前の方へ」
- 「自分の強みをうまく言葉にできていないと感じている事業者の方へ」
- 「外注は高い、でも自分では書けない、と思っている方へ」

## 10. 完成前チェックリスト

### FV（ファーストビュー）
- [ ] ヘッドラインを読んで「誰向けか」が3秒で分かるか
- [ ] 「何を得られるか」がFVだけで伝わるか
- [ ] FV内にCTAボタンがあるか
- [ ] 企業スローガン（「未来を変える」等）だけになっていないか

### ペイン・共感セクション
- [ ] 読者が使う言葉でペインを描いているか（自社目線の言い換えをしていないか）
- [ ] 「自分のことだ」と感じる場面・感情・行動が描かれているか
- [ ] あおりが3文以内に収まっているか

### ベネフィット・解決策
- [ ] 機能説明ではなく、顧客が得るものを書いているか（FAB変換済みか）
- [ ] 数字・具体性があるか
- [ ] ビフォアアフターのギャップが明確か

### 社会的証明
- [ ] 数字が具体的か（「多くの」ではなく「127社の」）
- [ ] 顔・名前・業種が入った声があるか
- [ ] 受賞・メディア掲載のバナーがFV付近にあるか

### CTA
- [ ] ボタン文言に「行動後の価値」が見えるか
- [ ] ボタン直下にリスク除去の一文があるか
- [ ] CTAが3箇所以上（FV・中盤・末尾）に配置されているか
- [ ] 「申し込む」「送信する」だけになっていないか

### リスク軽減
- [ ] 解約・無料期間・サポートが明示されているか
- [ ] FAQが「読者の不安リスト」として設計されているか

### 全体
- [ ] 「会社が言いたいこと」より「読者が知りたいこと」の順になっているか
- [ ] AI・専門用語が顧客に向けて不必要に出ていないか
- [ ] スマートフォンで見た時に1スクロールで要旨が分かるか
- [ ] ログイン済みユーザーに「無料で始める」を出していないか（ガードレール#9）
