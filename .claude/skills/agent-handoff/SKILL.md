# Agent Handoff

## 目的
引き継ぎ時の情報欠落を防ぎ、「何を見ればよいか分からない」状態をなくす。

## 使う場面

- `build` から `review` にレビュー依頼する時
- `arch` から `review` に評価依頼する時
- `ops` から `build` / `arch` に調査や修正を渡す時
- `growth` / `finance` から `lead` に判断を返す時

## 必須で含めること

1. 依頼の背景
2. 今回の目的
3. 見てほしい対象
4. すでに確認したこと
5. 未解決の論点
6. 次に必要な判断

## 出力フォーマット

```md
## Handoff

### 背景
- ...

### 目的
- ...

### 対象
- ファイル:
- 画面 / API / ドキュメント:

### すでに確認したこと
- ...

### 未解決
- ...

### 特に見てほしい観点
- ...

### 次に必要な判断
- ...
```

## 注意

- 「お願いします」だけで渡さない
- ファイル名だけでなく、何を見るべきかまで書く
- 自分の仮説と未確定事項を分ける
