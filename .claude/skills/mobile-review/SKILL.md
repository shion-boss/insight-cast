# mobile-review

新しい画面・コンポーネントを実装したあと、スマホで使えるレベルかを確認するチェックリスト。
実装完了後・コミット前に必ず実施する。

## いつ使うか

- 新しいページ・コンポーネントを作った直後
- 既存画面のレイアウトを変更した直後
- レビュアーへの依頼前のセルフチェックとして

## チェックリスト

### 1. レイアウト
- [ ] `h-screen` を使っていない（`h-dvh` を使う）
- [ ] テーブルをモバイルでそのまま出していない（`sm:hidden` カードリスト + `hidden sm:block` テーブルのデュアルレイアウトにする。`overflow-x-auto` + `min-w` だけでは横スクロールが残るので不可）
- [ ] `flex` の横並びがモバイルで縦に崩れていない（`flex-col sm:flex-row` パターン）
- [ ] ヘッダーはモバイルで「ロゴ（左）＋ハンバーガー（右）」のみ。タイトル・ボタン・アバターは PC（`lg`）以上にのみ表示する

### 2. タップターゲット
- [ ] すべてのボタン・リンクが `min-h-[44px]` 以上
- [ ] `text-xs py-1.5` など小さいボタンに `min-h-[44px]` を追加した
- [ ] リンクが文字だけの場合 `px-3 py-3` などでタップ領域を確保した

### 3. テキスト・コンテンツ
- [ ] 長いテキストが `truncate` または `line-clamp-N` で折り返し制御されている
- [ ] `whitespace-nowrap` が狭い幅で使われていない（はみ出す）
- [ ] フォントサイズがモバイルで読める大きさ（最低 `text-xs` = 12px）
- [ ] タブナビが 375px に収まっている（4タブ以上は `text-xs sm:text-sm` + `px-2.5 sm:px-3` に縮小）
- [ ] DB やAPIから取得したテキストをそのまま表示するとき、Markdown記法（`##`, `**`, `- ` 等）が混入していないか確認する。excerpt・プレビュー用途では必ずMarkdownをstrip する

### 4. フォーム・入力
- [ ] `<label>` と `<input>` が `htmlFor`/`id` で紐付いている
- [ ] 入力欄が `w-full` でモバイル幅に対応している
- [ ] `<select>` に `w-full` がある

### 5. インタラクション
- [ ] `role="dialog"` があればフォーカストラップを実装した
- [ ] `role="switch"` は `<button>` で実装した（`<div>` ではない）
- [ ] ハンバーガー・ドロワーは `h-dvh` を使っている
- [ ] ドロワーは右からスライドイン（`right-0` + `translate-x-full` → `translate-x-0`）で統一

### 6. 世界観（顧客向け画面のみ）
- [ ] キャラアイコンが主要な操作・通知に添えられている
- [ ] AI語（「生成」「プロンプト」「モデル」）が顧客向け画面に出ていない

## スクリーンショットによる確認方法

`npm run screenshot` を実行すると全54ページ（公開・認証・ツール・管理）をモバイル（375px）とPC（1280px）で撮影できる。
`.env.local` に `SCREENSHOT_EMAIL` / `SCREENSHOT_PASSWORD` を設定すると認証が必要なページも撮影される。

1. `npm run screenshot` を実行
2. `screenshots/<日付>/mobile/` の画像を目視確認
3. 上記リストを上から確認し、未対応項目を修正
4. 修正後に `screenshots/<日付>/desktop/` でPC表示が崩れていないことも確認

## 合格基準

すべての項目にチェックが入った状態でコミットする。
不明点はレビュアーに `mobile-review` スキルを指定して確認依頼する。

## 過去の主な修正事例（再発防止）

| 問題 | 原因 | 対処 |
|---|---|---|
| テーブルが横スクロール | `overflow-x-auto` + `min-w-[Npx]` だけで対処していた | `sm:hidden` カード + `hidden sm:block` テーブルのデュアルレイアウト |
| 設定タブが「セキュリ」で切れる | タブ4つが375pxに入りきらない | モバイルで `text-xs` + `px-2.5` に縮小 |
| 記事excerptに `## 見出し` が表示 | Markdown stripが `# ` のみで `##` 等に未対応 | 全Markdown記法を正規表現でstrip |
| ヘッダーにボタン・アバターが詰まる | モバイルでも同じヘッダーを出していた | ロゴ＋ハンバーガーのみ、他は `hidden lg:flex` |
| ドロワーが左から出る | `left-0` + `-translate-x-full` | `right-0` + `translate-x-full` に変更 |
