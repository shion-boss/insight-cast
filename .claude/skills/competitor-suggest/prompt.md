# 競合候補提案プロンプト

## 現行プロンプト（`competitor-suggestions/route.ts` 内）

```
以下の事業者の競合となりそうなウェブサイトを5件提案してください。

業種: ${industry ?? '不明'}
地域: ${location ?? '不明'}
自社HP: ${url ?? '不明'}

条件:
- 実在する可能性が高い具体的なURLを提案する
- 同じ地域・同業種の競合を優先する
- 全国チェーンより地域密着の競合を優先する
- 競合ごとに「何が特徴か」を1〜2文で簡潔に添える
- 必ずJSON配列形式のみで返す（説明文不要）

返答形式（このJSONのみ返す）:
[
  {
    "name": "競合名",
    "url": "https://example1.com",
    "summary": "地域密着で何を打ち出しているかの簡単な説明"
  }
]
```

---

## 各条件の設計意図

### 「実在する可能性が高い具体的なURL」

LLMは架空のURLを生成することがある。「実在する可能性が高い」という制約を入れることで、実際に存在するサイトに近いURLが出やすくなる。
ただし完全な保証はないため、UIでは「クリックして確認してください」と伝える必要がある。

### 「地域密着の競合を優先する」

全国チェーンのURLを提案しても、Insight Cast のユーザー（地域商圏の中小企業）にとって実用的な比較にならない。
同じ地域・同じ規模感で戦っている事業者との比較が、インタビューでの問いかけに活きる。

### `summary` フィールドを求める理由

競合URLだけ並べてもユーザーには「どれを選べばいいか」が分からない。
「〇〇が特徴の事業者です」という1〜2文の説明を添えることで、ユーザーが「これは知っている業者だ」「これを調べたい」と判断できる。

### 5件という提案数

3件だと選択肢が少ない。10件だと選ぶのが面倒。
5件が「選べる量」として適切。
実装側で `.slice(0, 5)` を設けてLLMが多く出しすぎた場合を制御している。

## 現行実装の限界と改善案

### 限界1: LLMの知識ベースに依存している

2024年以降に開業・廃業した事業者、URLが変更された事業者などは正確に出せない。
提案精度はLLMの学習データの時点に依存する。

**改善案A**: Google Places API / Google Custom Search API を使ってリアルタイムで検索する。精度は上がるが外部APIコストが発生するため、finance に試算依頼が必要。

**改善案B**: 提案URLをFirecrawlで事前確認し、200が返るURLだけユーザーに見せるバリデーションを追加する。

### 限界2: 自社HPの内容を読んでいない

現行は `url` を文字列として渡すだけで、HP内容を分析していない。
自社HPを読むことで「この業種・規模感・エリア」をより正確に把握した上で競合を提案できる。

**改善案**: `analyzeHp` でHP分析した結果の `current_content` を競合提案プロンプトにも渡す。HP分析と競合提案を連携させる。

### 限界3: 同業種かどうかの確認ができない

事業者が業種欄に「食品製造業」と入力しても、競合の `summary` に「食品製造」と書かれているとは限らない。
実際には全く別業種のURLが出てくることがある。

**改善案**: 提案後にユーザーが選んだURLを分析し、「業種が一致するか確認します」というステップを追加する。

## 改善版プロンプト（試案）

HP分析の結果を組み合わせる場合の改善案。採用前に人間のレビューが必要。

```
システムプロンプト:
あなたは地域密着の中小企業の競合調査を支援する専門家です。
全国チェーンより地域密着の競合を優先し、事業者が「知っているかもしれない」レベルの具体的な競合を提案します。
架空のURLを出さず、実在する可能性の高い候補だけを提案してください。

ユーザーメッセージ:
以下の事業者の競合候補を5件提案してください。

業種: ${industry}
地域: ${location ?? '不明'}
自社HP: ${url}
自社HPで伝えていること: ${currentContent.join('、')}

提案の優先順位:
1. 同じ地域・同業種の競合
2. 同程度の規模感（地域密着の中小企業）
3. 実在する可能性が高いURL

返答形式（このJSONのみ返す）:
[
  {
    "name": "競合名",
    "url": "https://example1.com",
    "summary": "何を特徴として打ち出しているかの説明（1〜2文）"
  }
]
```
