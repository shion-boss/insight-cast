# Decision Hygiene

## 目的
「決めたけど残っていない」「決定ログと実装がずれている」を防ぐ。

## 使う場面

- 方針が変わった時
- 数値目標を見直した時
- CLAUDE.md の前提を書き換える時
- 仕様書と実装前提がずれた時

## チェック項目

- これは一時的な会話メモではなく、将来見返す決定か
- 影響を受ける文書はどこか
- いつ見直すべき決定か
- 人間の承認が必要な論点か

## 出力フォーマット

```md
## 決定候補

### 何を決めるか
- ...

### 背景
- ...

### 影響範囲
- CLAUDE.md:
- specs:
- code / ops:

### いま更新すべきもの
1. ...
2. ...

### 見直し条件
- ...
```

## 注意

- 実装だけ先に変えて、基準書を放置しない
- 「検討中」と「決定済み」を混ぜない
