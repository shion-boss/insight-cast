# AI Education Loop

## 目的
プロダクトAIの改善を、勘ではなく症例ベースで回す。

## 使う場面

- `AIデザイナー` がキャラや裏方AIを改善する時
- `レビュアー` が教育課題を `AIデザイナー` に返す時
- `オペレーター` が現場で出た AI の弱さを整理する時

## 手順

1. 症例を 1件ずつ集める
2. 失敗を `prompt / context / rubric / structure / model / ui` に分類する
3. 最小の改善案を作る
4. `レビュアー` に渡す
5. 記録を残す

## 出力フォーマット

```md
## AI教育ケース

### 症例
- ...

### 何が弱かったか
- ...

### 原因分類
- prompt / context / rubric / structure / model / ui

### 改善仮説
- ...

### 変える場所
- ...

### review に見てほしいこと
- ...

### 次回の確認ポイント
- ...
```

## 注意

- 一度に大きく変えすぎない
- 教育担当が自分で合格判定までしない
- 「もっと自然に」ではなく症例ベースで書く
