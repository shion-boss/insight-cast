# ベネフィット設計ガイド

## なぜ「機能」を並べても売れないのか

読み手が買うのは「機能」ではなく「手に入る未来の自分」。
機能を並べるだけのコピーは **仕様書** であり、感情が動かない。

---

## FAB 分析：三層構造

```
Feature（機能）    : 何ができるか
Advantage（利点）  : 他と何が違うか / なぜそれが優れているか
Benefit（便益）    : 読み手の生活・感情に何が起きるか
```

### 変換の問い

Feature → Advantage：「だからどうなのか？（So what?）」
Advantage → Benefit：「それで読み手の生活がどう変わるか？」

### Insight Cast の FAB 変換例

| Feature（機能） | Advantage（利点） | Benefit（便益） |
|---|---|---|
| AIインタビュー | 話すだけで言語化される | 自分でゼロから考えなくていい |
| 月15回まで取材可能 | 週1〜2本のペースで更新できる | HP更新が習慣になる。止まらなくなる |
| 競合HP分析 | 何が足りないかが一目で分かる | 「何を書けばいいか」に迷わなくなる |
| 記事をブロック単位でコピー | コピペするだけで完成 | ブログ編集が1分で終わる |
| AIキャストが質問を考える | テーマ設定や準備が不要 | 「今日何を書こう」と悩まなくていい |

## 機能的ベネフィット vs 情緒的ベネフィット

### 定義

**機能的ベネフィット**：時間・コスト・スキル・手間に関わる具体的な得
**情緒的ベネフィット**：感情・自己イメージ・承認欲求・安心感に関わる主観的な得

### どちらを先に出すか

感情が先、理由が後。

> 感情で「動かし」、理由で「確信させる」

最初に情緒的ベネフィットで心を掴み、その後に機能的ベネフィットで論理的に後押しする。

### Insight Cast の 機能的 / 情緒的 ベネフィット一覧

| 機能的ベネフィット | 情緒的ベネフィット |
|---|---|
| 取材1回 = 約20分で完了 | 「今日も1本出せた」という小さな達成感 |
| 月15回の記事ネタが確保できる | HP更新に追われるストレスがなくなる |
| AIが質問を考えてくれる | 「準備しなくていい」という安心感 |
| 競合との差が可視化される | 「うちにはこれがある」という自信 |
| 記事がコピペできる状態で届く | 「自分の言葉で伝えられた」という誇り |

## 感情訴求 vs 論理訴求のバランス

### 商材タイプ別の基本比率

| 商材タイプ | 感情 | 論理 | 例 |
|---|---|---|---|
| 課題解決型（SaaS・サービス） | 50% | 50% | Insight Cast |
| ライフスタイル型（ファッション・高級品） | 70% | 30% | ブランド品 |
| 機能重視型（車・家電・保険） | 30% | 70% | ビジネスツール |
| 衝動購買型 | 80% | 20% | セール・限定品 |

### Insight Cast のセクション別バランス

```
Pain セクション    : 感情 70% / 論理 30%
                    → 読み手の痛みに寄り添い、深刻さを認識させる

Solution Bridge    : 感情 50% / 論理 50%
                    → 「これなら変われるかも」の期待と、根拠の両立

Feature/Benefit    : 感情 40% / 論理 60%
                    → 「なぜこのサービスなのか」の論理的な確信を作る

Social Proof       : 感情 30% / 論理 70%
                    → 事例・数字・声で信頼を固める

CTA                : 感情 60% / 論理 40%
                    → 最後の一押しは「今行動する理由」
```

### 感情訴求の4つのトリガー

1. **恐怖・損失回避**：「このままでは〇〇になる」
   - Insight Cast向け：「HPが止まったまま1年経つと、競合に差がつく」

2. **共感・承認**：「あなたの悩みは正しい。間違っていない」
   - Insight Cast向け：「書けないのはあなたのせいじゃない」

3. **理想・欲求**：「こうなれたら」という未来を見せる
   - Insight Cast向け：「今月も1本、自分の言葉で記事が出た」

4. **帰属・連帯**：「同じ悩みを持つ人たちがいる」
   - Insight Cast向け：「HP更新に悩む事業者が選んでいます」

### 論理訴求の3つの証拠タイプ

1. **数字**：「平均取材時間20分」「月15回まで」「1記事あたり¥83」
2. **権威**：「AIインタビューがE-E-A-T基準をクリアする理由」
3. **事例**：「外壁塗装業の山田さんは3ヶ月で8本の記事を出した」

## ベネフィットコピーの書き方 3ステップ

### Step 1: Feature を列挙する
まずサービスの機能を全部書き出す。評価しない。

### Step 2: 「だからどうなのか？」を3回繰り返す
各Featureに「So what?」を3回問い続けると情緒的ベネフィットに到達する。

**例**
```
Feature: AIがインタビューする
↓ So what?
Advantage: 話すだけで言語化される
↓ So what?
Benefit（機能的）: 自分でゼロから考えなくていい
↓ So what?
Benefit（情緒的）: 更新のプレッシャーから解放される
```

### Step 3: 40〜60代が「自分のことだ」と感じる言葉に変換する

| 書き手の言葉 | 読み手の言葉 |
|---|---|
| コンテンツ制作の効率化 | 記事を書く手間がなくなる |
| 自社の強みの言語化 | 自分には何があるか、初めて分かった |
| SEO対策コンテンツ | 検索で見つけてもらえるようになる |
| ナレッジの蓄積 | 話した内容が資産になっていく |

---

## やってはいけないこと

1. **Feature の羅列で終わる**：「AIインタビュー / 競合分析 / 記事生成」だけ並べる → 仕様書
2. **抽象的なベネフィット**：「ビジネス成長を支援」「可能性を広げる」→ 何も刺さらない
3. **感情訴求だけで根拠がない**：「あなたの未来が変わります！」→ 信頼が生まれない
4. **論理訴求だけで感情がない**：「当社のAIは業界最先端の〇〇技術を使用」→ 読まれない
5. **Before/Afterの場面がズレる**：Before「課題がある」After「課題が解決」→ 感情が動かない
