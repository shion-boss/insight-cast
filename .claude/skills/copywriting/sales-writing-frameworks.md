# セールスライティング フレームワーク集

## フレームワークを使う目的

フレームワークは「型」ではなく「読み手の感情の動線設計図」。
何を言うかより、**どの順番で言うか**が購買意欲を動かす。

---

## 主要フレームワーク 5選

### 1. AIDA / AIDMA / AIDCA

**構造**
```
AIDA  : Attention（注意）→ Interest（興味）→ Desire（欲求）→ Action（行動）
AIDMA : + Memory（記憶）を Desire の後に挿入
AIDCA : + Conviction（確信）を Desire の後に挿入
```

**使い分け**
| バリアント | 向いている場面 |
|---|---|
| AIDA | 汎用。衝動購買・短期接触が多いチャネル（SNS広告等） |
| AIDMA | ブランド認知を積み上げたい長期施策（メルマガ・ブログ） |
| AIDCA | 機能・ベネフィット中心の商材。信頼が必要な購買決定（SaaS・BtoB） |

**Insight Cast への適用**
AIDCA が最適。中小企業主が「本当に効くのか」という確信を持つまで購買しない。

```
A: 「更新が止まったHP、思い当たる節はありますか？」（注意）
I: 「書けないのは、やる気の問題ではありません」（興味）
D: 「AIキャストが毎月あなたの話を聞きに来ます」（欲求）
C: 「外壁塗装業・山田さんの事例：月3本の記事が3ヶ月続いた」（確信）
A: 「まず無料で1回体験する」（行動）
```

### 2. PAS / PASONA

**構造**
```
PAS   : Problem（問題）→ Agitate（扇動）→ Solution（解決）
PASONA: Problem → Agitate → SOlution → Narrow down（絞り込み）→ Action
```

**使い分け**
| バリアント | 向いている場面 |
|---|---|
| PAS | 短いコピー（SNS・見出し）。即購買につなげる場合 |
| PASONA | LPのボディコピー。課題解決型商材全般 |

**Agitate（扇動）の注意点**
- 痛みを深掘りして「深刻さを認識させる」ステップ
- やりすぎると「告訴状」になる（`lp-copy-principles.md` のNGパターン④参照）
- Insight Castの場合：読み手を責めず、**状況・方法・環境**を扇動する

**Insight Cast への適用例（PASONAでPainセクション設計）**
```
P: やる気があっても、問いがなければ始まらない。
A: 何を書けばいいか分からない / 強みが言葉にならない / 毎回ゼロから考えるのがたいへん
   → 「更新のたびにゼロから考える消耗が、続かない本当の理由です」
S: 問いさえあれば、答えられる。AIキャストが聞きに来ます。
O: 取材に答えるだけでいい人のための、月額プラン
A: まず1回、無料で体験してみる
```

### 3. BAB（Before-After-Bridge）

**構造**
```
Before : 読者の現状（痛み・不満）を具体的に描く
After  : 問題が解決した理想状態を見せる
Bridge : 現状から理想への橋渡しとして商品を提示
```

**強み**
- Before/After が「同じ場面」で描かれると最も力が出る
- 短く使えるので、セクション冒頭・SNS・メールに向く

**Insight Cast への適用例**
```
Before: 「今月も記事が書けないまま、もう月末だ」
After:  「今月も1本、自分の言葉で記事が出た」
Bridge: AIキャストが毎月取材に来るから、考えなくていい
```

**Before/After の場面ズレに注意**
- ✗ Before「コンテンツが不足している」→ After「記事が生成された」（抽象→機能説明でズレ）
- ✓ Before「また今月も書けなかった」→ After「今月も1本、届けられた」（同じ感情的場面）

---

### 4. QUEST

**構造**
```
Q: Question（問いかけで共感を誘う）
U: Unique（唯一つの利点に絞る）
E: Evidence（根拠・証拠を示す）
S: Story（顧客の成功物語で信頼を構築）
T: Trial（トライアルへ導く）
```

**向いている場面**
- LP全体の構成設計
- 「なぜInsight Castなのか」を伝えるセクション

**Insight Cast への適用例**
```
Q: 「書きたいのに書けない。それはネタがないからですか？」
U: 「Insight Castの強みは一つ：あなたが話した言葉をそのまま記事にすること」
E: 「平均取材時間20分。外壁塗装の山田さんは3ヶ月で8本の記事を出した」
S: 「山田さんは最初『自分には特別なことなんてない』と言っていた。でもミントが聞いた後、
    父から受け継いだ姿勢が記事になり、問い合わせが来るようになった。」
T: 「まず1回、カード不要で体験してみませんか」
```

## 「そうだったのか感」を生む洞察型コピーの技法

### 定義

読み手が「言われてみると確かに！」「ずっとモヤっていたことがやっと言語化された」と感じる瞬間を作るコピー。

記憶に残り、シェアされやすく、信頼が生まれる。

### 構造：前提→反転→着地

```
前提: 読み手が「当然そうだ」と思っている常識を提示する
反転: その常識をひっくり返す（または「実は違う」と示す）
着地: 本当の答え・洞察を端的に言い切る
```

**例1（認識の反転）**
```
前提: HPが更新されないのは、やる気がないからだ
反転: やる気はある。でも問いがない
着地: 問いさえあれば、答えられる
```

**例2（原因の再帰属）**
```
前提: 自分の強みが言葉にならないのは、強みがないからだ
反転: 強みはある。当たり前すぎて見えていないだけ
着地: 「当たり前」を聞き出すのがInsight Castの仕事
```

### 洞察型コピーを作る問い

1. 読み手はなぜ今できていないと**思っているか**（表面的な自己診断）
2. **本当の原因**は何か（構造・環境・方法の問題であることが多い）
3. 本当の原因を「言い切れる短い一文」にできるか

この3つを答えると「そうだったのか感」のある一文が生まれる。

## Insight Cast LP セクション別フレームワーク適用マッピング

| セクション | 推奨フレームワーク | 役割 |
|---|---|---|
| FV（ファーストビュー） | AIDCA の A-I | 注意を引き、興味を持たせる |
| Pain | PAS の P-A / BAB の Before | 問題を深掘りし、深刻さを認識させる |
| Solution Bridge | PAS の S / BAB の Bridge | 出口の予感を与え、次セクションへ誘う |
| Feature/Benefit | AIDCA の D-C / QUEST の U-E | 欲求を高め、確信させる |
| Social Proof | QUEST の S | 成功事例で信頼を構築する |
| Pricing/CTA | PASONA の N-A | 絞り込み＋行動を促す |

## フレームワーク選択の判断基準

| 状況 | 選ぶフレームワーク |
|---|---|
| LP全体の構成を設計する | AIDCA または QUEST |
| Painセクション単体を書く | PAS または BAB（Before部分） |
| SNS・広告の短いコピー | PAS（3行版） |
| 「なぜInsight Castか」を伝えるセクション | BAB または QUEST の U-E-S |
| 読み手に「そうだったのか」と感じさせたい箇所 | 洞察型コピー（前提→反転→着地） |
