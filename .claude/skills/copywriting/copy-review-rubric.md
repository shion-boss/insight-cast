# Copy Review Rubric

コピーを提出する前の **self-check**（マーケター用）と、
レビュアー がコピーを評価する時の **評価基準**（レビュアー用）を兼ねる。

---

## self-check の使い方（マーケター）

コピーを書いたら、提出前に以下の順で確認する。

1. **Step 1: 論証構造を確認**（セクションA）
2. **Step 2: 感情の流れを確認**（セクションB）
3. **Step 3: リズムと呼応を確認**（セクションC）
4. **Step 4: 言葉選びを確認**（セクションD）
5. **Step 5: やってはいけないパターンを確認**（セクションE）

全項目に ✅ が付けられたら提出する。
一つでも ❌ があれば `lp-copy-principles.md` を参照して書き直す。

---

## セクションA：論証構造

| # | チェック項目 | ✅ / ❌ |
|---|---|---|
| A-1 | コピーに「問いへの答え」があるか（命題の羅列になっていないか） | |
| A-2 | 使っている構造が「A否定→B提示」「対比」「AでもBでもなくC」のいずれかであるか | |
| A-3 | 「AでもBでもなくC」構造を同一セクション内で2回以上使っていないか | |
| A-4 | 否定しているのは「方法・環境」であって「読み手本人」ではないか | |
| A-5 | Painセクションの締めが「出口の予感」で閉じられているか（痛みだけで終わっていないか） | |

---

## セクションB：感情の流れ

| # | チェック項目 | ✅ / ❌ |
|---|---|---|
| B-1 | 感情の流れが「否定→共感→解放」の順になっているか | |
| B-2 | 「共感」パートで、読み手の感情・状況・行動を読み手の言葉で描いているか | |
| B-3 | 「解放」は最後の1〜2文に収まっているか（2文以上続けていないか） | |
| B-4 | 読み手を「告訴」していないか（「怠った」「不足している」「損をしている」を主語として使っていないか） | |
| B-5 | Painセクションが痛みの列挙だけで終わっていないか | |

---

## セクションC：リズムと呼応

| # | チェック項目 | ✅ / ❌ |
|---|---|---|
| C-1 | セクション内のopeningとclosingで同じモチーフ・構文を呼応させているか | |
| C-2 | LPのFVヘッドラインとクロージングCTAのコピーが呼応しているか | |
| C-3 | 長文と短文が交互に現れるリズムになっているか（長文だけ・短文だけが連続していないか） | |
| C-4 | 接続詞（そして/また/しかし/さらに）の多用がないか（段落ごとに1つ以内が目安） | |

---

## セクションD：ターゲット層への言葉選び

| # | チェック項目 | ✅ / ❌ |
|---|---|---|
| D-1 | 「PCは使えるけどAIは初めての55歳の工務店社長」が3秒で意味を取れるか | |
| D-2 | 禁止語（プロンプト・モデル・トークン・生成・アルゴリズム・コンバージョン・KPI・エンゲージメント）が顧客向け文面に含まれていないか | |
| D-3 | カタカナ・横文字に「= 〜のこと」という言い換えが添えられているか（省略した場合、省略可能な根拠があるか） | |
| D-4 | トーンが「ビジネスライクすぎず、砕けすぎず」の適切な距離感か | |
| D-5 | 概念語（価値・差別化・言語化・戦略・ソリューション）を2つ以上同一文に並べていないか | |

---

## セクションE：やってはいけないパターン

| # | チェック項目 | ✅ / ❌ |
|---|---|---|
| E-1 | 二重否定を使っていないか（「〜でないわけでもない」等） | |
| E-2 | 「AでもBでもなくC」の多用・入れ子構造になっていないか | |
| E-3 | openingで作った世界観がclosingで事務的に終わっていないか | |
| E-4 | 「そこでInsight Castです」のように、感情を飛ばして唐突にサービスを出していないか | |
| E-5 | 全体が「会社が言いたいこと」の順になっていないか（「読者が知りたいこと」の順か） | |

---

## レビュアー 向け評価基準

self-check の ✅/❌ に加えて、以下の観点で総合評価をつける。

### 論証構造スコア

| スコア | 基準 |
|---|---|
| 3（強い） | 構造が明確。A否定→B提示 または 対比 が機能しており、読み手の「認識の更新」が起きる |
| 2（普通） | 構造はあるが弱い。列挙に近い。読み手の認識は揺れるが更新まで届かない |
| 1（弱い） | 構造が見えない。言いたいことを並べているだけ |

### 感情の流れスコア

| スコア | 基準 |
|---|---|
| 3（強い） | 否定→共感→解放 が自然に流れている。読み手の感情変化を追える |
| 2（普通） | 共感はあるが解放が弱い、または解放はあるが共感が薄い |
| 1（弱い） | 感情設計がない。告訴型または感情無視の論理展開 |

### リズムスコア

| スコア | 基準 |
|---|---|
| 3（強い） | opening と closing が呼応している。長短のリズムが心地よい |
| 2（普通） | 呼応は弱いが読めるリズム。接続詞の多用が一部あるが許容範囲 |
| 1（弱い） | openingとclosingに関係がない。または長文の塊が続いて読み疲れる |

### 言葉選びスコア

| スコア | 基準 |
|---|---|
| 3（強い） | 40〜60代の非IT・中小企業主の言葉で書かれている。概念語がない |
| 2（普通） | 一部に横文字・概念語があるが言い換えが添えられている |
| 1（弱い） | ターゲット層が3秒で意味を取れない言葉が複数ある |

---

## 総合評価の出し方

各スコアを合算する（最高12点）。

| 合計点 | 評価 | 対応 |
|---|---|---|
| 10〜12 | 高品質 | 世界観チェックのみで公開可 |
| 7〜9 | 要改善 | 2点以下のスコアの項目を `lp-copy-principles.md` 参照で修正 |
| 4〜6 | 再起草 | 論証構造か感情設計のどちらかが根本的にやり直し |
| 3以下 | 却下 | 全体を構造から書き直す。フレームワーク選定から再スタート |

---

## レビュアー 向け出力フォーマット

```
## コピーレビュー: <対象セクション・コピー名>

### self-check 未通過項目
- [ ] A-x:
- [ ] B-x:
（省略可。全通過なら「self-check: 全項目通過」と記載）

### スコア
- 論証構造: 3 / 2 / 1
- 感情の流れ: 3 / 2 / 1
- リズム: 3 / 2 / 1
- 言葉選び: 3 / 2 / 1
- 合計: xx / 12

### 総合評価
高品質 / 要改善 / 再起草 / 却下

### 最も強い箇所
> 引用

### 最も弱い箇所と改善案
1. 問題: / 改善案:
2. 問題: / 改善案:

### マーケター へのフィードバック
（次回に同じ弱点を繰り返さないための一言）
```
