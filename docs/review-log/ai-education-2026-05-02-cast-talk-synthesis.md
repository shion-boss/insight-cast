### 何が弱かったか

**弱点1（症例A対応）: 実績・数値に関する禁止事項が設計に存在しない**
Cast Talk 生成プロンプト（`conversation-prompt.md`）の禁止事項は「専門用語・機械的表現・AI丸出しの表現」が中心。「架空の実績を語らない」「数値は根拠ある場合のみ」という制約がなかった。世界観ガードは充実しているが、情報の正確性ガードが欠けていた。

**弱点2（症例B対応）: インタビュー形式の冒頭設計が「構造」のみ**
`conversation-prompt.md` の【インタビュー形式の場合】には「インタビュアー側の発言は短め（問いを立てる）」という構造指示のみ。「なぜその問いを立てるのか」「インタビューの入り口として読者に状況を伝える」という内容設計がなかった。

**弱点3（症例C対応）: 締め方の「良さ」の定義がない**
【会話の雰囲気】に「余韻で終わる方が自然」という記述はあるが、「読者が何を持ち帰れるか」を含む締め方の基準がなかった。また Cast Talk 会話生成プロンプトに相槌の指示がなく、インタビュアー用プロンプトの「相槌の例」が参照されていなかった。

### 何が弱かったか

**弱点1（症例A対応）: キャラごとの「やってはいけない語尾」がプロンプトに書かれていない**
`CAST_TALK_VOICE` の `prohibited` フィールドには `['だよね', 'じゃん', ...]` という全員共通の崩し語のみ。各キャラが「他のキャラの語尾」に侵食されないための固有禁止語がない。
`conversation-prompt.md` の【口調の基本ルール】も「それぞれの語尾パターン」を示すのみで「〇〇は△△を使わない」という形式になっていない。

**弱点2（症例B対応）: ターン数指定が「守ってほしい」ではなく「そうなりやすい」書き方になっている**
「会話は8〜14ターン」という記述は情報提供に見えるが、強制力が弱い。特に「14ターンを超えたら締めを作る」という安全弁がない。短すぎる場合も「最低8ターン」という明示的な要件になっていない。

**弱点3（症例C対応）: 「結論を先に言わない」が禁止事項の文脈のみで、冒頭の設計意図として書かれていない**
禁止リストは「やってはいけない」の列挙であり、モデルが「なぜいけないか」を理解できる設計になっていない。
「冒頭2ターンは問いかけ・観察・きっかけを投げる。答えの形にしない」という積極的な設計として書き直す方が効果的。

### 改善仮説

**仮説1（症例A: 口調混在 → `context` + `prompt`）**

`conversation-prompt.md` の【口調の基本ルール】に、キャラごとの「使わない語尾」を1行追加する。

変更前（現状）:
```
- ミント: 一人称「私」。柔らかい敬語。「〜ですよね」「〜が多いんです」「〜かなと思います」
- クラウス: 一人称「私」。落ち着いた敬語。断定より問いかけ。「〜ということですよね」「〜なんでしょうね」
- レイン: 一人称「僕」。軽めの敬語・観察スタンス。「〜じゃないですか」「〜が気になって」
```

変更後（案）:
```
- ミント: 一人称「私」。「〜ですよね」「〜が多いんです」「〜かなと思います」。クラウスのような断定問いかけ（「〜ということですよね」）を使わない。
- クラウス: 一人称「私」。断定より問いかけ。「〜ということですよね」「〜なんでしょうね」。ミントのような共感語尾（「〜ですよね」単体の相槌）で締めない。
- レイン: 一人称「僕」。「〜じゃないですか」「〜が気になって」。クラウス・ミントのような「〜ですよね」「〜なんでしょうね」で締めない。観察・整理が語尾に出る。
```

同時に `lib/characters.ts` の `CAST_TALK_VOICE` の `prohibited` を更新し、キャラごとに「他のキャラの代表語尾」を追加する。

**仮説2（症例B: ターン数逸脱 → `structure`）**

`conversation-prompt.md` の【全体のルール】のターン数記述に「最低・最大」の強制表現を追加する。

変更前:
```
- 会話は8〜14ターン。読み切れる長さを守る
```

変更後（案）:
```
- 会話は8〜14ターン。最低8ターンは書く（短すぎると山場が生まれない）。14ターン目までに必ず締めに入る（続けても良い話でも14で着地させる）。
```

**仮説3（症例C: 結論先出し → `prompt`）**

`conversation-prompt.md` の【全体のルール】または【会話の雰囲気】セクションに、冒頭の積極的設計を追記する。禁止事項の「結論を先に言いすぎる」との連携を意識して書く。

追加案（【全体のルール】の末尾に追加）:
```
- 冒頭2ターン（ターン1と2）は「問いかけ」か「観察の共有」のみ。答えの形・教訓の形にしない。「〜ということですよね」「〜が大事なんですよね」という結論的な締め方はターン3以降に使う。
```
