### 1. Hero セクション

#### 現在の実装（`app/page.tsx`）
- H1: 「ホームページの更新、手伝います。」
- サブコピー: 「AIキャストが取材に来て、あなたの話から記事の素材を作ります。答えるだけで、ホームページが少しずつ育っていきます。」
- CTA（未ログイン）: 「無料で取材を体験する →」
- サブCTA: 「キャストを見る」

#### 評価
H1は分かりやすいが「手伝います」だとサービスの輪郭がぼやける。
「AIキャスト」という言葉を前に出してほしいが、現状サブコピーに埋まっている。
メインコピー「あなたにとっての当たり前は、まだ伝わっていない価値かもしれません。」が使われていない。

#### 改善案A（メインコピーを前に出す案）
```
バッジ: AI取材サービス

H1:
あなたにとっての当たり前は、
まだ伝わっていない価値かもしれません。

サブコピー（H1直下）:
Insight Cast の動物インタビュアーが取材に来て、
あなたの話からホームページの素材を作ります。
答えるだけで大丈夫です。

CTA（未ログイン）: まずは無料で取材を体験する →
サブCTA: キャストを見る
```

#### 改善案B（現行の分かりやすさを活かしつつ補強）
```
バッジ: AI取材サービス

H1:
ホームページが止まっているなら、
AIキャストが取材に来ます。

サブコピー:
動物モチーフのインタビュアーが質問するので、
答えるだけで記事の素材が届きます。
「何を書けばいいか分からない」を解消します。

CTA（未ログイン）: カード不要・無料で体験する →
サブCTA: キャストを見る

メインコピーをHero内の補足として追加:
小テキスト:
「あなたにとっての当たり前は、まだ伝わっていない価値かもしれません。」
```

#### 推奨案
改善案B。理由: 40-60代が3秒で「何のサービスか」を理解できることを優先。
メインコピーは補足として残し、感情的な引き込みに使う。

### 2. 課題提起セクション

#### 現在の実装
3カードで「何を書けばいいか分からない」「取材・撮影が大変」「自社の強みを言葉にできない」を表示。

#### 評価
3つの課題は適切。ただしカード同士に関係性がなく、共鳴しにくい。
「あなたのことですか?」という問いかけが弱い。

#### 改善案コピー

```
セクション見出し:
「HPが止まっているのは、意欲がないからじゃない。」

リード文（見出し直下）:
更新が後回しになるのには、理由があります。

カード01:
「何を書けばいいか分からない」
ブログを書こうと思うたびに、「今さら何を？」と手が止まる。
ネタがないのではなく、ネタの見つけ方が分からないのです。

カード02:
「取材・撮影が大変で後回しになる」
更新のたびに写真を撮って文章を考えて。
それだけで半日かかる。本業が忙しいのに、続くわけがない。

カード03:
「自社の強みを言葉にできない」
「特別なことなんて何もない」——そう感じていませんか。
でも、それが一番伝わっていない価値かもしれません。
```

### 3. 解決策セクション（3ステップ）

#### 現在の実装
「たった3ステップで、記事の素材が届く」として実装済み。内容はほぼ適切。

#### 評価
Step 02「AIが取材する」の「専門知識なしで話すだけ」は分かりやすい。
ただしStepがサービスの「やること」で終わっており、「あなたが楽になる」への橋渡しが弱い。

#### 改善案コピー

```
セクション見出し:
やることは、ただ「答えるだけ」。

リード文:
ネタ出しも、文章作りも、Insight Cast が引き受けます。

Step 01: HPを分析する
取材先のホームページと競合を登録するだけで、
「今のHPで何が足りないか」「どこを強化すべきか」が整理されます。
何を取材すればいいかが、最初から見えた状態で始められます。

Step 02: AIキャストが取材する
ミント、クラウス、レインが質問します。
答えるだけでOK。資料も整った言葉も必要ありません。
約20分のチャットで、一次情報が引き出されます。

Step 03: 記事の素材が届く
取材内容をもとに、ブログや実績ページに使える文章の素材を作ります。
「何を書くか」に迷う前に、素材がある状態を作ります。
```

### 4. キャラクター紹介セクション

#### 現在の実装
`/cast#[id]` へのリンクつきカードで3名分（ミント・クラウス・レイン）を表示。portrait 画像、名前、label、description、specialtyを表示。

#### 評価
実装は十分だが、「なぜキャラが3人いるのか」「何が違うのか」の説明が弱い。
キャラの選び方に迷うユーザーへのガイドがない。

#### 改善案コピー

```
セクション見出し:
あなたを担当するキャスト

リード文:
3名の無料キャストがいます。
「誰に頼もうか迷う」場合は、ミントから始めてください。

[キャラカード — 現行実装を活かしながら補足テキストを追加]

ミントの補足（card内追加文）:
はじめての方にもっともおすすめのキャスト。
お客様目線で「ホームページを見る人が知りたいこと」を引き出します。

クラウスの補足:
「なぜあなたの仕事は他と違うのか」を論理的に掘り下げます。
業種や専門知識に詳しくない方でも大丈夫。むしろ客観的な目線が強みです。

レインの補足:
「どう訴求するか」「何が差別化になるか」をマーケティング視点で聞きます。
集客や問い合わせにつなげたい方に向いています。

[追加キャスト エリア]
ラベル: 近日公開のキャスト
補足テキスト: ハル（写真から取材）、モグロ（はい/いいえで掘り下げ）、コッコ（宣伝・告知）
```

### 5. 社会的証明セクション

#### 現在の実装
なし（ドッグフーディング段階のため未作成）。

#### Phase 2 での表現戦略

顧客事例がまだない段階で「社会的証明」を偽装することはしない。
代わりに以下2つの表現でカバーする。

**A. 自社ドッグフーディングの開示**

```
セクションラベル: From the Team

見出し:
私たち自身が、Insight Cast を使っています。

本文:
Insight Cast は今、自社のホームページ更新を
Insight Cast 自体で進めています。
このブログもその過程から生まれています。

自分たちが「ホームページ更新が止まっている状態」を経験し、
それを解決するためにこのサービスを作りました。
実際に使いながら改善してきた過程をブログで公開しています。
現在も取材を続けながら、このサービスを育てています。

CTA: 運営ブログを読む →  [/blog へのリンク]
```

**B. 「取材ログ例」による疑似証明**

すでに実装されているアウトプット例セクション（外壁塗装の工務店の会話ログ + 記事素材）が社会的証明の代替として機能する。このセクションの見出しに「実際の取材の流れ」というコンテキストを明示することで信頼感を補う。

```
セクションラベル: Output Example

見出し（改善案）:
こんな取材をして、こんな素材が届きます
（実際の取材の流れより）

補足テキスト（追加）:
これは実際の取材の流れを再現した例です。
専門的な準備は何もいりません。このように会話するだけです。
```

**C. ブログ記事の実績カード（中間マイルストーン達成後に更新）**

中間マイルストーン（ブログ10本以上）達成後に、公開済み記事数・取材回数などの数値を
「Insight Cast の取材実績」として表示するセクションを追加する。
現段階では準備のみ。

### 6. 料金セクションへの導線

#### 現在の実装
3プランをLP内に直接表示している。「まず無料で、気軽に始められます」という見出し。

#### 評価
プラン表示はあるが「なぜ無料から始めるべきか」の説得力が弱い。
無料 → 有料のアップグレード動機がLPから見えない。

#### 改善案コピー

```
セクション見出し:
まず無料で体験してください。月2回まで、カード不要で使えます。

リード文:
使いやすいかどうかは、使ってみないと分かりません。
カード不要、メールアドレスだけで今すぐ始められます。

[プランカード — 現行実装を活かす]

無料プランカードの補足テキスト追加:
「体験してみて、続けたいと思ったらプランを選んでください。
途中で変更・解約はいつでもできます。」

有料プランへのアップグレード動機（個人向けカード内）:
「月2回で足りなくなった方へ。月10回まで取材でき、
競合調査も3社対応します。週1〜2本のペースで更新したい方に向いています。」
```

### 7. CTA・問い合わせセクション

#### 現在の実装
ミントのアイコン + 「迷ったらまず聞いてください」 + 「まず相談してみる →」ボタン（/contact へ）

#### 評価
キャラアイコンがあって温かい。コピーも十分。ただし「何を相談できるのか」がやや曖昧。

#### 改善案コピー

```
[ミントのアイコン — 現行実装を活かす]

見出し:
「自社に使えるか」から聞いてください。

本文:
業種のことも、ホームページの状況も、
何も準備しなくて大丈夫です。
「こんな使い方はできますか?」「まず何から始めればいいですか?」
そんなところから始めましょう。

CTA: 問い合わせる（無料・返信は1営業日以内） →
```

## 優先対応リスト（Phase 2 の中間マイルストーン向け）

Phase 2 でLPに問い合わせを呼び込むために、以下の順で対応を検討する。

### 優先度 高（まず対応）
1. **Hero H1 の改善** — メインコピーをHeroに組み込む（改善案B推奨）
2. **社会的証明セクションの追加** — ドッグフーディング開示（パターンA）を /blog への導線として追加

5. **キャラカードへの「選び方ガイド」追加** — 「迷ったらミントから」の一文
6. **料金セクションのリード文改善** — 「1回だけ体験して」の文言

### 優先度 低（ブログ本数が増えてから）

## 現行実装の良い点（変えない方がいいもの）

- Hero の数値バー（3名・約20分・¥0）: 信頼感を補う。維持
- アウトプット例の会話ログ + 記事素材の2カラム: サービスを一番具体的に伝えている。維持
- 最終CTAセクションのミントアイコン: キャラが世界観を担っている。維持
- 比較表（ChatGPT・自前・Insight Cast）: 競合との差を客観的に整理できている。維持
- Heroの2次CTAボタン「キャストを見る」: /cast への誘導として機能。維持

## reviewer へのチェック依頼

以下の点について世界観チェックをお願いします。

1. **課題提起の見出し案「HPが止まっているのは、意欲がないからじゃない。」** — ユーザーを責めているように読めないか
2. **ドッグフーディング開示のコピー** — 「自分たちが使っています」という表現が信頼に働くか、逆に「未完成感」を与えないか
3. **「まず無料で体験してください。月2回まで、カード不要で使えます。」** — 無料プランの仕様（月2回）を正しく伝えつつ、始めやすさが伝わるか
4. **キャラ補足テキスト** — キャラの個性と一致しているか（arch にも確認依頼を検討）
