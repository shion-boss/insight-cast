# オーケストレーターのガバナンス明文化

- 作成日: 2026-05-04
- 決定者: ディレクター + ユーザー

---

## 決定内容

オーケストレーター（Sonnet / Claude Code のトップレベル会話）から Edit / Write 権限を剥奪し、役割を「対話・指示出し・結果統合」の3点に限定する。

### 権限変更の内容

**剥奪した権限:**
- `Edit`（ファイルの部分編集）
- `Write`（ファイルの新規作成・上書き）

**残した権限:**
- `Read`（ファイル読み込み）
- `Bash`（シェルコマンド実行 — git・確認系に限定して使う）
- `Task`（サブエージェントへの委任）

`.claude/settings.json` の `permissions.allow` を空配列 `[]` に変更することで技術的に強制する。

### オーケストレーターに許可する操作（3点のみ）

1. ユーザーとの対話・意思確認・選択肢提示
2. ディレクターをはじめとする社内AIへの指示出し
3. 複数エージェントの結果統合・ユーザーへの報告

---

## 理由

オーケストレーターが直接 Edit / Write を行った結果、以下の問題が多発していた。

- 実装の一部が漏れる（設計の全体像を把握せず局所的に書いた）
- レビュアーを通さずに機能が混入する
- エンジニア / AIデザイナーとの役割境界が曖昧になる
- 「自分でやった方が速い」という判断が委任原則を形骸化させる

役割分担を **文書で規定するだけでなく技術的に強制する** ことで、再発を防ぐ。

---

## 承認フロー（新規定）

| 変更対象 | 必要な承認 |
|---|---|
| ディレクター以下のルール（`.claude/agents/*.md`、`.claude/skills/` 等） | オーケストレーターの同意 |
| オーケストレーター自身のルール（CLAUDE.md の委任原則セクション） | ディレクター + ユーザーの承認 |
| `permissions.allow` の変更（権限の復元・追加） | ディレクター + ユーザーの承認 |

オーケストレーターは自分自身の権限を単独で復元できない。

---

## 関連変更ファイル

- `CLAUDE.md` — 委任の原則セクションに「できること / できないこと / ルール変更の承認フロー」を追記
- `.claude/settings.json` — `permissions.allow` を `["Edit","Write"]` から `[]` に変更
