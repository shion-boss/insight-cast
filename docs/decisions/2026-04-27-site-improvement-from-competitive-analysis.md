# 決定: 競合調査結果をもとにした公開サイト改善方針と優先順位

## 日付

2026-04-27

## 背景

マーケター による競合調査（`docs/market/competitive-analysis-2026-04.md`）の結果、以下の核心が明らかになった。

1. 「初心者・低リテラシー層 × 一次情報を引き出せる」の空白ポジションに直接競合なし
2. 全競合の共通弱点は「材料を自分で持っている人向け」設計であること
3. AI検索時代に一次情報の価値が上がる構造的追い風がある
4. リスクは Castmagic 日本語強化・ChatGPT インタビュアーモード強化の可能性

これを受け、Phase 2（ドッグフーディング実証中）における公開サイトの改善方針を整理・記録する。

現在のサイト構成:
- `/` トップ（Hero・サービス概要・キャラ紹介・料金・FAQ）
- `/service` サービス詳細
- `/cast` AIキャスト一覧
- `/pricing` 料金
- `/about` 会社概要
- `/blog` ブログ
- `/faq` FAQ
- `/contact` お問い合わせ

制約:
- Phase 2 中: ドッグフーディング実証が最優先、エンジニアリングコストは最小化
- 改善の優先順位: コピー変更 > 構成変更 > 機能追加
- ターゲット: 40-60代のAI未経験の中小企業主

---

## 改善すべきページの優先順位（最大3ページ）

### 優先1位: `/` トップページ

**理由:**

- コンバージョン（「無料で始める」）の最大の入口
- 競合との差別化訴求が最も効く場所
- 現状のトップページは構成・コンテンツともに充実しているが、「なぜ Insight Cast だけが引き出せるのか」の説明が弱い
- 競合調査で明確になった空白ポジション（低リテラシー層 × 一次情報引き出し）の訴求が現状では間接的

具体的な不足:
- Hero 以降のページ内に「なぜAIツールやライターに頼んでも一次情報が出てこないのか」の説明がない。差別化比較（⑪ Compare）はあるが、「引き出しの設計が違う」という根拠が薄い
- 「AIを使っている感覚を持たせない」という設計思想の価値が、訪問者に伝わる仕掛けが少ない
- ドッグフーディングの「From the Team」セクション（⑬）はあるが、「使った結果どうなったか」の実績や変化がまだ薄い

### 優先2位: `/blog` ブログ（コンテンツ追加）

**理由:**

- 中間マイルストーン（ブログ累計10本）の達成がドッグフーディング実証の核
- ブログは「Insight Cast が語るべきテーマ」で検索流入・AI検索参照の両方に効く
- 現在5本で、10本まであと5本。競合調査で見えた「語るべきテーマ」が明確になったため、今すぐコンテンツ戦略を固めるべきタイミング
- ブログページ自体の構成は機能しているため、テーマ選定とコンテンツ制作が主な施策

### 優先3位: `/service` サービスページ

**理由:**

- 「引き出す設計」の詳しい説明を置くべき場所だが、現状は「3ステップで届く」のプロセス説明に留まっている
- 競合との本質的な差異（材料がない人向け設計）を深く理解したい人がたどり着くページ
- 「なぜ動物キャラが来るのか」「なぜ構えないのか」の設計思想を説明するセクションがない
- コピー・文章の追記で対応可能であり、エンジニアリングコストが小さい

## 各ページで変えるべき観点

### 優先1位: トップページ（`/`）

#### コピー

**現状の課題箇所:**

1. Compare セクション（⑪）のヘッダー: 「他の方法と、比べてみてください。」
   - 比較表の列が「AIツールで書いてもらう」「自分や社員で書く」「Insight Cast」になっている
   - 「AIツールで書いてもらう」は競合調査で見えた最大の差別化ポイントだが、表が機能チェックに留まっており「なぜ AIツールでは一次情報が出ないのか」という根拠説明がない
   - 表の前に1〜2文の根拠文を入れることで説得力が大きく上がる

2. Workflow セクション（④）のリード: 「ネタ出しも、文章作りも、Insight Cast が引き受けます。」
   - 実態に近いが「ネタ出し」という表現が「ネタがない」前提に見える
   - 「あなたの話の中に、まだ言葉になっていない価値がある」という前提に変えた方がターゲットに刺さる

3. Pain セクション（②）見出し: 「HPが止まっているのは、意欲がないからじゃない。」
   - 強い言葉だが「では何なのか？」の答えが各カードの説明文に留まっている
   - 見出し直下の補足「更新が後回しになるのには、理由があります。」を「何を書けばいいかが分からない。それが根本の原因です。」のように「一次情報が言語化できていない」に直結させると、後の解決策との一貫性が高まる

#### 構成・セクション追加

- Compare セクションの前に「なぜ Insight Cast だけが引き出せるのか」の根拠として、1〜2文の説明を加える（セクション追加ではなく既存セクション内の文章追加で対応可）
- 「From the Team」セクション（⑬）に、実際にドッグフーディングで得られた成果（記事本数・取材回数など）を数字で追記する。現状は「使っています」の宣言だが、「N本の記事を取材で作った」等の実績があると信頼感が上がる

#### 訴求軸の変更

- 現状: 「答えるだけで記事が届く（手間の削減）」が主軸
- 変えたい方向: 「あなたが当たり前と思っていることに、まだ伝わっていない価値がある（一次情報の価値）」を前面に出す
- 変更の優先度: 高い。競合調査が示した空白ポジションがここに一致する
- 「手間削減」は入口として引き続き使う。その先に「実は価値が埋まっている」という発見体験を設計する

### 優先2位: ブログ（コンテンツ追加）

**競合調査で見えた「Insight Cast が語るべきテーマ」:**

| テーマ | 優先度 | 競合調査との対応 |
|---|---|---|
| 「AIに書いてもらった記事と、取材で引き出した記事は何が違うのか」 | 高 | ChatGPT/Jasper系との差別化。AI検索時代に一次情報が有利な理由 |
| 「HPが更新できていない理由は、意欲ではなく"ネタの見つけ方"がないから」 | 高 | 記事代行系競合との差別化。「書く代行」でなく「引き出す設計」の説明 |
| 「自社で取材したら、こんな話が出てきた（ドッグフーディング記録）」 | 高 | 実証コンテンツ。ツールのデモとして機能する |
| 「なぜ動物キャラにインタビューしてもらうのか」 | 中 | 設計思想の説明。「構えさせない装置」の価値を事業者向けに語る |
| 「AI検索時代に、中小企業のHPはどう変わるべきか」 | 中 | 構造的追い風の説明。将来の必要性を先手で語る |
| 「業種ごとのHP更新の詰まりポイント（建設・飲食・士業など）」 | 低（Phase 4 以降） | 業種特化は Phase 4 方針のため、後送り |

**ブログのコンテンツ方針:**

- 「思想・哲学・なぜやるか」系の記事を先行させる。SEO記事より先に、「このサービスを使う意味が分かる記事」を作る
- ドッグフーディングの記録記事（代表者取材から何が出てきたか）を最優先の1本目の候補にする
- AI検索でも引用される形式（一次情報 + 独自の視点）を意識する

### 優先3位: サービスページ（`/service`）

#### コピー追加

現状の STEP 02「AIキャストが取材する」の段落に、以下の趣旨の文を追加:

「AIツールは、あなたが素材を持っていることが前提です。Insight Cast は逆で、素材を引き出すことが仕事の核心です。」

これは現状の説明文にない視点であり、競合との根本差を端的に表す。

#### 構成

- サービスページに「なぜ動物キャラが取材するのか」の理由説明セクション（短い）を追加する
  - 現状: 動物キャラの紹介はトップページにある。サービスページでは「チャット形式」の説明に留まっている
  - 追加方向: 「動物インタビュアーは"構えない"のです」という設計思想の1段落
  - エンジニアリングコスト: テキスト追加のみ

## やらないこと

以下は競合調査から浮かぶが、Phase 2 では手を出さない。

| 施策 | やらない理由 |
|---|---|
| 業種別のLP・コンテンツ作成（建設向け、飲食向け等） | Phase 4 以降の方針。業種特化は汎用実証が先 |
| SEO最適化（キーワード選定・内部リンク設計） | 記事本数10本が揃ってから。今は一次情報の蓄積が先 |
| SNS運用の強化（Twitter/Instagram等） | Phase 2 は LP・ブログ基盤の確立が優先。SNSは基盤があってからの増幅 |
| 料金ページの大幅改変 | 現在の価格設定・構成は固定費モードと整合。構造を変える理由がない |
| 動画コンテンツの制作 | コスト・時間が大きく Phase 2 に合わない。ブログが機能してから検討 |
| A/Bテストの導入 | セッション数が500未満のうちはサンプルサイズが足りない |
| 競合比較の詳細ページ（Insight Cast vs Castmagic 等） | 競合名を出すのはリスクが大きい。差別化はポジティブな訴求で行う |

## 影響を受ける場所

- `app/(site)/page.tsx`: Compare セクション・Pain セクション・From the Team セクションのコピー変更
- `app/(site)/service/page.tsx`: STEP 02 の説明文追加、設計思想セクションの追加
- `ops/weekly-review.md`: 来週の施策として記録
- ブログコンテンツ: `docs/` ではなく実際のブログ記事として Supabase に追加

## 見直し条件

- Phase 2 進捗を月1回確認し、流入源がブログか直接かで重点施策を調整する
- ブログ10本達成時点で改めて訴求軸を見直す
- 重大な競合参入（Castmagic 日本語強化等）が確認された場合は即時見直し
