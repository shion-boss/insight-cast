# インタビュー充足判定条件改善に伴う トークンコスト試算

## 改善案の概要

### 改善前（現在）
`lib/characters.ts` の `SUFFICIENCY_INSTRUCTION`:
```
十分と判断する条件:
- 事業者のこだわり・工夫・お客様への価値のいずれかが1点以上出ている
```
**特徴**: 抽象的な回答 でも「価値が1点出ている」と判定 → 終了提案 → **平均8ターン で終了**

### 改善後（提案）
条件追加：
```
具体的な価値がまだ1点も出ていない場合（例: 回答が抽象的・短く、
具体的なエピソードやこだわりが出ていない）は、引き続き続けてください
```
**特徴**: 抽象的な回答のみでは充足と認めない → 追加質問 → **最大12ターンまで延伸する可能性**

## トークンコスト試算ベース

### 前提条件

**システムプロンプト + コンテキスト（固定）:**
| 項目 | トークン数 |
|---|---|
| システムプロンプト（キャラ別） | ~2,000 |
| 品質ガイド・テーマ・調査結果 | ~1,500 |
| **小計（毎ターン固定）** | **~3,500** |

**会話データ（可変）:**
| 内容 | トークン/ターン |
|---|---|
| 事業者の回答 | 100-150 |
| AIの質問 | 50-80 |
| **ターンあたり合計（入力）** | **150-230** |
| **AIの返答（出力）** | **~150** |
| **ターンあたり総トークン** | **300-380** |

**モデル**: claude-sonnet-4-6
| | 金額 |
|---|---|
| Input | $3 / 1M tokens |
| Output | $15 / 1M tokens |

### 計算式

```
1ターン全体のコスト = (固定入力 + 可変入力) × ($3/1M) + 出力 × ($15/1M)

固定入力: 3,500 tokens
可変入力: 190 tokens (平均)
出力: 150 tokens

= (3,500 + 190) × 0.000003 + 150 × 0.000015
= 3,690 × 0.000003 + 150 × 0.000015
= $0.01107 + $0.00225
= $0.01332 / ターン
≈ $0.0135 / ターン (切り上げ)
```

## 代替案・軽減方法

### 案1: プロンプト最適化による効率化
- システムプロンプトをより簡潔にして 固定入力 3,500T → 2,800T に削減
- **効果**: ターンあたり約 ¥0.0010 削減 → 月100件時に ¥1,000 コスト削減可能

### 案2: キャッシュ活用（Claude API Prompt Caching）
- 固定部分（システムプロンプト + 品質ガイド）を キャッシュ化
- **効果**: キャッシュヒット時の入力コスト を 80% 削減
- **実装**: 2026-05月中に検討可能

### 案3: ターン数上限の厳しい制御
- 改善案の「12ターン上限」を 「10ターン上限」に変更
- **効果**: 平均10ターンに制限できればシナリオC相当でコスト確定

## 参考：プロンプト最適化による削減シミュレーション

### 最適化案（例）

**現在のシステムプロンプト削減箇所:**
- キャラごとの反応フレーズを、完全リスト記載から「バリエーションを保つこと」の原則記載に縮約
- CAST_TALK_VOICE の Cast Talk 定義は実装時のみ参照で、インタビュー プロンプトには含めない

**削減効果:**
```
現在: 約2,000 tokens (キャラプロンプト)
削減後: 約1,400 tokens (フレーズ例を削減)

1ターン入力削減: 600 tokens × $0.000003 = $0.0018
月100件時削減: $0.0018 × 100 = $0.18 ≈ ¥27/月

小さく見えるが，年単位（12ヶ月 × ¥27 = ¥324）や
月インタビュー数増加時（200件なら ¥54）には効果あり
```

実装は **低優先度** だが，Phase 3 の強め モードに向けて 2026年6月までに検討推奨。
