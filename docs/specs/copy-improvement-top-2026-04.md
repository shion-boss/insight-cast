## 変更方針（マーケターによる整理）

競合調査が明確にした Insight Cast の唯一の差別化:
「材料を自分で持っていない人向けの設計」「引き出すこと自体が商品の核」

現状トップページは「手間削減」訴求が主軸になっており、この差別化が間接的にしか伝わっていない。
3箇所のコピー改善で「引き出す設計が違う」という根拠を前面に出す。

**コピー全体の判断基準:**
「PCは使えるけどAIは初めての55歳の工務店社長が読んで、3秒で何のことか分かるか」で判定している。

## 変更箇所1: Compare セクション 前置き文

### 変更前

```
見出し: 他の方法と、比べてみてください。
前置き文: （なし）
```

### 変更後

```
見出し: 他の方法と、比べてみてください。
前置き文:（見出しの直後に以下を追加）

AIツールも、記事代行も、「すでに書く材料を持っている人」向けに作られています。
Insight Cast はその前の段階——「何が自社の強みか、まだ言葉にできていない」という
根本から引き出すことが、仕事の核心です。
```

### 変更の意図

- 「AIツールが使える人向け」「引き出しの機能がない」という競合共通の弱点を、相手を名指しせずに端的に示す
- 「材料がない人向け」という空白ポジションの説明を、比較表を見る前に置く
- 「仕事の核心」という言い方で、他との本質的な違いを強調
- 40-60代に伝わる日常語（「材料」「根本」「引き出す」）を使用。専門用語なし

## 変更箇所2: Pain セクション 補足文

### 変更前

```
見出し: HPが止まっているのは、意欲がないからじゃない。
補足文: 更新が後回しになるのには、理由があります。
```

### 変更後

```
見出し: HPが止まっているのは、意欲がないからじゃない。
補足文: 「何を書けばいいか分からない」——それが、根本の理由です。
```

### 変更の意図

- 現状の補足文「理由があります」は、読者に「その理由とは？」の疑問を持たせるが、答えを引き延ばすだけで一次情報訴求につながらない
- 「何を書けばいいか分からない」はペインカード01のタイトルと重複しているが、補足文でこそ先に答えを明示することで「ああ、まさに自分だ」という共感を早く作れる
- 「根本の理由」という言い方で、「意欲や時間ではなく、もっと深い原因がある」という流れを自然に引き継ぐ
- 直後の3枚のカードが「何を書けばいいか分からない」「取材が大変」「強みを言葉にできない」と並ぶ中で、補足文がカード01の内容とつながる構造になる

### 世界観セルフチェック

- [x] アイコンで世界観を担い、テキストは分かりやすさに徹しているか（UI）
  - Pain セクションはテキスト主体のセクション。分かりやすさ優先で適切
- [x] 読み物文脈の言い換えが過剰でないか（LP・ブログ）
  - 「根本の理由」は日常語。過剰な言い換えなし
- [x] 40-60代に伝わるか
  - 「何を書けばいいか分からない」は読者の実際の言葉に近い。「コンテンツ不足」などの表現は使っていない

## 変更箇所3: From the Team セクション 信頼訴求文

### 変更前（現状の2段落）

```
（段落1）
このサービスは、私たち自身が「HPを更新しなきゃ、でも何を書けばいいか分からない」
という状態を経験したことから生まれました。

（段落2）
Insight Cast は今、自社のホームページ更新をこのツールで進めています。
このブログの記事も、実際に取材を重ねながら作っています。
売り込みではなく、私たちが先に使って確かめたことを、そのままお届けしています。
```

### 変更後（変更は段落2のみ。段落1は変更なし）

```
（段落1）
このサービスは、私たち自身が「HPを更新しなきゃ、でも何を書けばいいか分からない」
という状態を経験したことから生まれました。
（変更なし）

（段落2・変更）
Insight Cast は今、自社のホームページ更新をこのツールで進めています。
代表者への取材を6回重ね、そこから生まれたブログ記事がすでに5本あります。
売り込みではなく、私たちが先に使って確かめたことを、そのままお届けしています。
```

### 変更の意図

- 「実際に取材を重ねながら作っています」という宣言文を、「6回・5本」という数字に置き換える
- 「このブログの記事も」という漠然とした指示を削除し、具体的な実績として明示
- 競合調査が示した「早期に実証実績を積み先行資産化する」方針に直結する変更
- 段落1は現状のままにし、サービス誕生の経緯（共感の文脈）を維持する

### 更新タイミングの注意

取材回数・ブログ本数が増えたタイミングで数字を更新すること。
現在値: 取材6回・ブログ5本（2026-04-27時点）

次の更新推奨タイミング:
- ブログ10本達成時（中間マイルストーン達成直前）
- 取材10回達成時
